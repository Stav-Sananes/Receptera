<objective>
Build the two service-wrapper modules that sit between the chunker (Plan 04-02) and
the ingest+retriever (Plan 04-04): the BGE-M3 embedding client and the ChromaDB
collection wrapper. Both follow the Phase 3 LLM client pattern — pure async wrappers
with typed errors, no global state, no logging in the hot path beyond fail-fast init.

**Module 1: `receptra.rag.embeddings`** — wraps `ollama.AsyncClient` for BGE-M3
embeddings (RESEARCH §Cluster 2 + §BGE-M3 Pattern):
- `BgeM3Embedder.create_and_verify()` async classmethod: instantiates `AsyncClient(host=settings.ollama_host)`, calls `client.show("bge-m3")` to fail-fast if the model is not pulled, returns the embedder. Raises `RagInitError(code='model_missing', detail="...run: make models-bge")` if `show` fails. Plan 04-05 calls this from FastAPI lifespan.
- `embed_one(text: str) -> list[float]`: returns the 1024-dim vector. Calls `client.embed(model='bge-m3', input=text, keep_alive='5m')` and unwraps `resp.embeddings[0]`.
- `embed_batch(texts: Sequence[str], batch_size: int | None = None) -> list[list[float]]`: chunks the list into slices of `settings.rag_embed_batch_size` (default 16) and calls `client.embed(input=batch_list, ...)`. Ollama processes the input list as a true batch (RESEARCH §Cluster 2 verified). Returns list of 1024-float lists in input order.
- `MODEL = "bge-m3"`, `DIM = 1024` class-level constants exposed for tests + Plan 04-05 health endpoint.
- `keep_alive = "5m"` (RESEARCH §Cluster 2: not `-1` since BGE-M3 is only 1.2 GB and embed is fast; 5m amortizes within an ingest job + query session without parking VRAM forever)

**Module 2: `receptra.rag.vector_store`** — wraps `chromadb.HttpClient` for the
single `receptra_kb` collection (RESEARCH §Cluster 1 + §ChromaDB Pattern):
- `COLLECTION_NAME = "receptra_kb"` module constant (Plan 04-04 ingest + Plan 04-05 routes import this).
- `parse_chroma_host(host_url: str) -> tuple[str, int]`: splits the full URL form (`http://chromadb:8000`) Phase 1 already pinned in `settings.chroma_host` into `(hostname, port)`. Uses `urllib.parse.urlparse` (stdlib). Raises `RagInitError(code='chroma_unreachable')` on malformed URL.
- `open_collection() -> Collection`: calls `parse_chroma_host(settings.chroma_host)` → `chromadb.HttpClient(host, port)` → `client.heartbeat()` (raises on unreachable; wrapped to `RagInitError(code='chroma_unreachable')`) → `client.get_or_create_collection(name=COLLECTION_NAME, metadata={"hnsw:space": "cosine"})` (idempotent — chromadb 1.5+ returns existing without overwriting metadata, per RESEARCH §Cluster 1 citing chroma-core/chroma migration.mdx).
- Sync client + `asyncio.to_thread` wrapping in routes (Plan 04-05 + 04-04). The wrapper itself is sync because that's the simpler upstream contract; async-ification happens at the consumer.

Cosine distance lock: `metadata={"hnsw:space": "cosine"}` is the v0.4-compatible
legacy form (RESEARCH §Cluster 1: "Use legacy `metadata=` form — it survives across
all 1.x versions and has zero risk of being silently rejected if ChromaDB introduces
a `configuration_json` parameter rename in a patch release"). BGE-M3 emits L2-normalized
vectors per its model card, so cosine distance maps directly to `1 - cos_sim`.

```python
class BgeM3Embedder:
    DIM = 1024
    MODEL = "bge-m3"

    def __init__(self, client: AsyncClient) -> None:
        self._client = client

    @classmethod
    async def create_and_verify(cls) -> "BgeM3Embedder":
        client = AsyncClient(host=settings.ollama_host)
        try:
            await client.show(cls.MODEL)
        except Exception as e:
            raise RagInitError(code="model_missing", detail=...) from e
        return cls(client)

    async def embed_one(self, text: str) -> list[float]:
        resp = await self._client.embed(model=self.MODEL, input=text, keep_alive="5m")
        return list(resp.embeddings[0])

    async def embed_batch(self, texts: Sequence[str], batch_size: int = 16) -> list[list[float]]:
        out: list[list[float]] = []
        for i in range(0, len(texts), batch_size):
            batch = list(texts[i : i + batch_size])
            resp = await self._client.embed(model=self.MODEL, input=batch, keep_alive="5m")
            out.extend(list(v) for v in resp.embeddings)
        return out
```

       ```python
       """BGE-M3 embedder via Ollama (RAG-01).

       1024-dim L2-normalized dense vectors for Hebrew + multilingual text.
       Pattern mirrors receptra.llm.client (Plan 03-03): typed errors,
       fail-fast classmethod, no global state, no logging beyond fail-fast init.

       Plan 04-05 lifespan calls BgeM3Embedder.create_and_verify() once at
       startup; consumers (Plan 04-04 ingest + retriever) hold the singleton
       on app.state.embedder.
       """
       from __future__ import annotations

       from collections.abc import Sequence

       from ollama import AsyncClient

       from receptra.config import settings
       from receptra.rag.errors import RagInitError


       class BgeM3Embedder:
           """BGE-M3 (1024-dim, cosine-normalized) embedder backed by Ollama."""

           DIM = 1024
           MODEL = "bge-m3"
           _KEEP_ALIVE = "5m"  # RESEARCH §Cluster 2: not -1; amortizes intra-job calls without parking VRAM

           def __init__(self, client: AsyncClient) -> None:
               self._client = client

           @classmethod
           async def create_and_verify(cls) -> "BgeM3Embedder":
               """Construct + fail-fast if bge-m3 is not pulled.

               Plan 04-05 lifespan calls this; failure raises RagInitError with
               actionable detail ("run: make models-bge") so the FastAPI startup
               banner tells the developer how to recover.
               """
               client = AsyncClient(host=settings.ollama_host)
               try:
                   await client.show(cls.MODEL)
               except Exception as e:
                   raise RagInitError(
                       code="model_missing",
                       detail=f"Ollama model {cls.MODEL!r} not available. Run: make models-bge",
                   ) from e
               return cls(client)

           async def embed_one(self, text: str) -> list[float]:
               """Single-text embed. Returns 1024 floats."""
               resp = await self._client.embed(
                   model=self.MODEL, input=text, keep_alive=self._KEEP_ALIVE,
               )
               return list(resp.embeddings[0])

           async def embed_batch(
               self,
               texts: Sequence[str],
               batch_size: int | None = None,
           ) -> list[list[float]]:
               """Batched embed. Defaults batch_size to settings.rag_embed_batch_size (16)."""
               bs = batch_size if batch_size is not None else settings.rag_embed_batch_size
               out: list[list[float]] = []
               for i in range(0, len(texts), bs):
                   batch = list(texts[i : i + bs])
                   resp = await self._client.embed(
                       model=self.MODEL, input=batch, keep_alive=self._KEEP_ALIVE,
                   )
                   out.extend(list(v) for v in resp.embeddings)
               return out


       __all__ = ["BgeM3Embedder"]
       ```

    2. **backend/src/receptra/rag/vector_store.py** — implement open_collection +
       parse_chroma_host verbatim from RESEARCH §ChromaDB Pattern (lines 427-465):

       ```python
       """ChromaDB collection wrapper (RAG-02).

       Sync HttpClient (D-03 lock) + asyncio.to_thread at the consumer site
       (routes / ingest). The receptra_kb collection is the single-tenant v1
       store; v2 multi-tenant adds tenant_id metadata filtering on top of this.

       Plan 04-05 lifespan calls open_collection() once at startup; consumers
       (Plan 04-04 ingest + retriever) receive the Collection via app.state.
       """
       from __future__ import annotations

       from urllib.parse import urlparse

       import chromadb
       from chromadb.api.models.Collection import Collection

       from receptra.config import settings
       from receptra.rag.errors import RagInitError

       COLLECTION_NAME = "receptra_kb"


       def parse_chroma_host(host_url: str) -> tuple[str, int]:
           """Split 'http://chromadb:8000' into ('chromadb', 8000).

           Required because Phase 1 (settings.chroma_host) pinned the FULL URL
           form; chromadb.HttpClient takes (host, port) separately. Defaults
           port to 8000 when the URL omits an explicit port.
           """
           parsed = urlparse(host_url)
           if not parsed.hostname:
               raise RagInitError(
                   code="chroma_unreachable",
                   detail=f"Invalid CHROMA_HOST: {host_url!r}",
               )
           return parsed.hostname, parsed.port or 8000


       def open_collection() -> Collection:
           """Idempotent collection open with fail-fast heartbeat.

           RESEARCH §Cluster 1: get_or_create_collection in chromadb 1.5+
           returns the existing collection without overwriting metadata if it
           already exists, so this function is safe to call at every backend
           startup (Plan 04-05 lifespan does exactly that).

           Cosine distance lock: metadata={'hnsw:space': 'cosine'} matches
           BGE-M3's L2-normalized output. Legacy form chosen over
           configuration={'hnsw': {'space': 'cosine'}} per RESEARCH §Cluster 1
           ("survives across all 1.x versions").
           """
           host, port = parse_chroma_host(settings.chroma_host)
           try:
               client = chromadb.HttpClient(host=host, port=port)
               client.heartbeat()  # raises on unreachable
           except RagInitError:
               raise
           except Exception as e:
               raise RagInitError(
                   code="chroma_unreachable",
                   detail=f"ChromaDB not reachable at {host}:{port}: {e}",
               ) from e
           return client.get_or_create_collection(
               name=COLLECTION_NAME,
               metadata={"hnsw:space": "cosine"},
           )


       __all__ = ["COLLECTION_NAME", "open_collection", "parse_chroma_host"]
       ```

       Note the inner `except RagInitError: raise` clause: parse_chroma_host can
       itself raise `RagInitError` and we don't want the broad `except Exception`
       to wrap it twice (loses the original code).
