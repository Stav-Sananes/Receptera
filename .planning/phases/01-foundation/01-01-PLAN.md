<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| repo → git history | Anything committed leaks forever. Secrets (.env) and model weights (multi-GB) must never enter history. |
| repo → docker build context | Files inside `.dockerignore`-excluded paths are still added to build context unless blocked; weights + secrets must be excluded or they end up in image layers. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01-01 | Information Disclosure | `.env` file with user-local config or secrets | mitigate | `.gitignore` includes `.env`, `.env.local`, `.env.*.local` patterns; `.env.example` is the only env template that ships. |
| T-01-01-02 | Information Disclosure | Model weight files (multi-GB GGUF / CT2 / safetensors) | mitigate | Both `.gitignore` and `.dockerignore` explicitly block `*.bin`, `*.safetensors`, `*.ct2`, `*.gguf`, `*.pt`, `*.pth` so weights cannot be committed or baked into images. Weights live in `~/.receptra/models/` (user home, per research §2). |
| T-01-01-03 | Tampering | LICENSE text authenticity | mitigate | Apache 2.0 verbatim text from apache.org, acceptance criteria grep for canonical section headers. No modifications beyond the copyright line. |
| T-01-01-04 | Information Disclosure | User-uploaded knowledge base (future Phase 4) committed by accident | mitigate | `.gitignore` contains `knowledge/*` with `!knowledge/.gitkeep` and `!knowledge/sample/` exceptions; only sample docs ship with the repo. |
| T-01-01-05 | Information Disclosure | GitHub accidentally renders Hebrew content with bidi rendering bugs exposing structure | accept | Research §6.2 confirmed GitHub markdown renderer has known mixed-direction bugs; mitigation is two separate files (README.md EN, README.he.md HE). No sensitive content in either file — public-facing docs only. |
</threat_model>
