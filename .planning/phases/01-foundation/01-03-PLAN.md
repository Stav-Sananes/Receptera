<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser ↔ Vite dev server | Vite HMR websocket; trusted in dev only. Production build has no dev server. |
| browser ↔ backend via proxy | Vite dev proxy forwards `/api/*` and `/ws/*` to localhost:8080. In production the reverse proxy is done by whatever serves the static bundle (future Phase). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-03-01 | Tampering | Vite dev server binds 0.0.0.0 | accept | Required for Docker container exposure. Dev-only; production serves the static `dist/` bundle, not the dev server. `strictPort: true` prevents silent port drift. |
| T-01-03-02 | Information Disclosure | Source maps shipped in build output | accept | `sourcemap: true` in `vite.config.ts` aids debugging; the v1 product is OSS so sources are public anyway. Not a threat. |
| T-01-03-03 | Supply Chain | npm deps introduce non-permissive license transitively | mitigate | Plan 06 adds `license-checker --onlyAllow` to CI. This plan only pins versions; enforcement lives in the CI plan. `license-checker@^25.0.1` is listed in devDependencies here so the CI check can run. |
| T-01-03-04 | Elevation of Privilege | `eslint-plugin-react-refresh` + HMR in production build | mitigate | `npm run build` excludes `react-refresh` (it's a dev-only Babel transform). Confirmed by ESLint config scoping it under `react-refresh/only-export-components` warn — no runtime behavior in bundle. |
| T-01-03-05 | Information Disclosure | RTL + Hebrew rendering bugs could expose mis-structured content to users | accept | Phase 1 ships only an empty placeholder page; there's no user-generated or sensitive Hebrew content. Phase 6 addresses bidi rendering properly via `hebrew-rtl-best-practices` skill. |
</threat_model>
